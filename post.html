30-Day Data Science Challenge

Today I designed and started my challenge to become a better data scientist. In the next 30 days, I will study at least one data analysis article online. The first 15 days will include learning from Kaggle's articles. In the beginning, I will just read and rewrite the codes from many of the popular analyses. I know I will not be able to understand them all, but the main purpose here is to immerse myself in some of the best techniques out there. Remember when I came to the U.S. and started to learn English. But how did I become an English speaker? I immersed myself in talks from radio and TV. I remember the time when I put on my headphones, closed my eyes and listened to some politic radio talk shows before going to bed at night. I could not understand a thing. But by immersing myself in English, I improved quickly and become proficient in a short period of time. Learning data science is a lot easier since I already know a lot. But this technique can still be applied and I expect to see good results in no time. LET'S GET STARTED!!!

The progress will be updated on Github HERE

To make to process faster and help me achieve this goal easier, I will only give a brief update everyday about what I learn and what I need to learn more on.

30-Day Data Science Challenge - Day 1

The first analysis I learned today was the chocolate ratings dataset. The data provides multiple features such as company, country of origin, cocoa contains, etc. that affect the ratings of these chocolate products.

What I learned
  • Review matplotlib commands
  • Review some basic pandas operations such as groupby, value_counts, changing data types, modifying columns.
  • Learn new techniques to modify and replace strings in colum names
  • Learn how to deal with messy data with misspelled and redundant values.
  • Review seaborn plots.
  • Learn how to use crosstab and aggregated data to get more insights
  • Learn how to investigated some unknown feature by looking at its correlation to other features

What I need to improve on
  • Learn more about heatmap
  • Learn more about how to split data with multiple countries into individual data
